# Create PV, PVC and then deployment.
- **Create a pv**
	- Name is `tiger-pv`
	- Size: `1Gi`
	- Policy: `retain`
	- Mode: `ReadOnlyMany`
- **Create a pvc**
	- Name is `tiger-pvc`
	- Size same as pv
	- Project name is `page`
	- Mode same as pv
- **Create an Deployment with**
	- Name of deployment is `tiger`
        - consume the pvc `tiger-pvc`
   	- Mount pvc to `/usr/share/nginx/html`
	- Image is  `registry.ocp4.example.com:8443/redhattraining/hello-world-nginx:v1.0`
	- Application uses this link to show output http://test-anishrana2001.apps.ocp4.example.com
	- After attaching storage it shows desired output
---

## Open the web console 
```
oc whoami --show-console
```
1. Create the PV Manifest (tiger-pv.yaml)
apiVersion: v1
kind: PersistentVolume
metadata:
  name: tiger-pv
spec:
  capacity:
    storage: 1Gi
  accessModes:
    - ReadOnlyMany
  nfs:
    server: 172.17.0.2
    path: /tmp
  persistentVolumeReclaimPolicy: Retain
  storageClassName: slow
  volumeMode: Filesystem
copy
Apply the PV:

oc apply -f tiger-pv.yaml
copy
2. Create the PVC Manifest (tiger-pvc.yaml)
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: tiger-pvc
  namespace: default
spec:
  accessModes:
    - ReadOnlyMany
  resources:
    requests:
      storage: 1Gi
  storageClassName: slow
  volumeMode: Filesystem
copy
Apply the PVC:

# First delete any existing PVC with the same name
oc delete pvc tiger-pvc

# Then apply the new PVC
oc apply -f tiger-pvc.yaml

✔️ This will bind immediately
